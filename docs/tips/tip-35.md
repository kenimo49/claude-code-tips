# Tip 35: Be braver in the unknown; iterative problem solving

> 未知に勇敢に、反復的に問題解決

**Category**: Workflow
**Related**: [Tip 3: Break down problems](./tip-03.md), [Tip 32: Abstraction levels](./tip-32.md)

## Overview

Claude Codeを使うようになってから、未知の領域でもより勇敢になれるようになった。

## Real Examples

### React Expertise Not Required

[Daft](https://github.com/Eventual-Inc/Daft)で働き始めた時、フロントエンドの問題を発見。Reactの専門家ではないが、掘り下げることにした。

結果：反復的に問題を解決する方法を知っていたので解決できた。

### Unknown Territory: Rust + Python

cloudpickleがGoogle Colab + Pydanticで動かない問題、Python + RustでJupyterLabでの表示問題。Rustは初めて。

イシューを作成して他のエンジニアに任せることもできたが、掘り下げることにした。

## The Process

1. Claude Codeが初期解決策を提案
2. うまくいかない場合、ペースを落とす
3. 別の方向を試す
4. 速度を調整：探索時は速く、詳細確認時は遅く
5. 「この行は具体的に何を意味する？」と質問

## The Lesson

未知の領域でも、Claude Codeを使えば思った以上のことができる。
