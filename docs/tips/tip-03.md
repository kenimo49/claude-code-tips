# Tip 3: Break down large problems into smaller ones

> 大きな問題を小さな問題に分解して、確実に解決する

**Category**: Workflow
**Related**: [Tip 12: Invest in your workflow](./tip-12.md), [Tip 39: Planning and prototyping](./tip-39.md)

## Overview

最も重要な概念の1つ。従来のソフトウェアエンジニアリングと同じ原則がClaude Codeにも適用される。

## The Concept

直接A→Bに行く代わりに：

```
A ──────────────────────────────> B
```

段階的に進む：

```
A ──> A1 ──> A2 ──> A3 ──> B
```

## How to Apply

1. Claude Codeが一発で解決できない場合、問題を分解するよう依頼
2. 個々の部分を解決できるか確認
3. まだ難しければ、さらに小さなサブ問題に分解
4. すべて解決可能になるまで続ける

## Real Example

音声認識システムの構築：

1. まず、モデルをダウンロードするだけの実行ファイルを作成
2. 次に、音声を録音するだけの実行ファイルを作成
3. 録音済み音声を文字起こしする実行ファイルを作成
4. 最後にすべてを統合

## Why This Matters

- 問題解決スキルとソフトウェアエンジニアリングスキルは依然として重要
- Claude Codeはこれらのスキルを適用することでより強力になる
